# Tcpdump Capture Analysis

## Capture command

sudo tcpdump -i any -c 100 -w capture.pcap

## Number of packets

104 packets captured.

## Observed protocols

Protocols observed in capture:

- ARP
- IP

No DNS or HTTP traffic observed.

## Explanation

The capture was performed inside WSL environment.  
WSL network stack may only capture traffic generated by the Linux subsystem.

If no DNS queries or web requests are generated from WSL, tcpdump will only capture minimal background traffic.

## Security perspective

Low traffic captures like this are common when analyzing isolated environments.

To generate more traffic during capture we can run commands like:

ping google.com
curl google.com
nslookup google.com

This gerenates more traffic and more captured packets
